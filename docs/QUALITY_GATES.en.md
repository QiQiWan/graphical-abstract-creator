# Quality Gates

Q0 Content brief sufficiency: block PPTX generation if the graphical abstract content brief is absent. Strict generation requires a content sufficiency score of at least 5/8.

Q1 Claim provenance: every visible claim must come from user input, uploaded source material, or a clearly marked assumption. Strict mode requires `claim_source` for every block.

Q2 Visual story coherence: one central claim, one dominant visual path, no dashboard-like clutter, and no unsupported mechanistic embellishment.

Q3 Editable-vector construction: all text must be text boxes and all visuals should be native PowerPoint shapes.

Q4 Typography and language: concise labels, no paragraphs, controlled bilingual hierarchy, and Chinese top-journal wording discipline when applicable.

Q5 Palette and contrast: one approved palette, restrained semantic colors, readable text contrast, and no casual mixing of unrelated palettes.

Q6 PowerPoint technical purity: no media folder, picture objects, OLE objects, audio/video, or external linked assets.

Q7 Package completeness: deliver PPTX plus requested spec, notes, and audit report.

Q8 Palette preview: when the user is choosing a style, provide or generate an editable vector palette-strip preview.
