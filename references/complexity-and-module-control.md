# Complexity and Module Control

## Four-module cap
A graphical abstract may contain at most four main modules. Five or more main modules make the image too complex for journal display and must be merged.

Allowed module roles:
- input/source;
- method/model;
- mechanism/process;
- validation/result;
- application/decision.

Because there are five possible roles but only four allowed modules, merge at least one pair when all five appear. Recommended merges:
- input + method when the source data directly enters the model;
- mechanism + result when the mechanism explains the measured outcome;
- result + application when the result directly supports decision or deployment;
- method + mechanism when the method is itself the mechanism.

## Complexity floor
The preview must not be too simple. It must contain:
- one dominant visual core;
- at least three distinct scientific visual object types;
- at least one mechanism cue, such as transformation, coupling, constraint, pathway, interaction, feedback, scale bridge, or causal arrow;
- at least one result or evidence cue when a result is provided;
- supporting context without turning into a poster.

## Complexity ceiling
The preview must not be overcrowded. It should contain:
- no more than four main modules;
- no more than six primary arrows or connectors;
- no more than two short visible labels per module;
- no more than one bottom key message;
- no decorative legends unless essential.

If the content exceeds the ceiling, move details to the PPT notes, caption, or quality report.
