# Preview Review Standard

Review the generated preview before any PPT is built. Reject the preview if any hard gate fails.

## Hard gate 1: visual complexity
The preview must be visually richer than a simple block diagram. It must include one dominant core object, supporting scientific objects, meaningful arrows or interactions, and a coherent visual scene. A row of equal rectangles with text fails.

## Hard gate 2: mechanism completeness
The preview must show how the main scientific process works. It should make the transition from input/source to method/mechanism to result/application visible. If the content brief lacks one part, mark it as absent rather than inventing it.

## Hard gate 3: large/small frame hierarchy
The preview must have a clear hierarchy. One central or primary frame should dominate. Supporting frames should be smaller, quieter, and secondary. Equal-sized card grids fail unless the user explicitly asks for a comparison table.

## Hard gate 4: information-density lower bound
The preview must contain enough scientific information to remain meaningful when scaled down in a paper. It should not rely on generic icons, empty space, or vague labels. It should include concrete objects, mechanism cues, and at least one result or evidence cue when available.

## Hard gate 5: academic aesthetics
The preview must look like a publication figure. It should be aligned, restrained, balanced, legible, and non-cartoonish. Avoid flashy gradients, stock icon collages, UI dashboard styling, excessive badges, and decorative clutter.

## Hard gate 6: module complexity
The preview must contain no more than four main modules. If more than four modules appear, merge modules and regenerate the preview.
