# Publication-Grade Graphical Abstract Standard

Use this standard for strict graphical abstract generation and review.

## Story standard

- Use one central visual claim.
- Preserve only elements that support the claim.
- Every block must have a semantic role: input, method, mechanism, result, validation, or application.
- Avoid turning the figure into a project-management workflow unless the paper itself is a workflow paper.
- Do not show journal-style labels, quality labels, or internal workflow labels as visible figure content.

## Composition standard

- Apply `references/composition-and-compactness.md` before building the PPTX.
- Prefer one large core frame plus smaller support frames.
- Keep 3-5 primary modules by default.
- Use active visual coverage of roughly 68-78% and quiet whitespace of roughly 18-28%.
- Title band should remain compact; bottom key message should be light and short.
- Avoid equal-size card rows, excessive bullets, and large bordered conclusion boxes.

## Visual standard

- Use alignment grids and consistent spacing.
- Use native PowerPoint vector shapes only.
- Use restrained colors and strong hierarchy.
- Use callouts only for verified results or key mechanisms.
- Avoid crowded labels, heavy gradients, shadows, clip-art aesthetics, and unstructured icon collages.

## Typography standard

- Use editable common fonts.
- Use no more than two font families.
- Use font size and weight before extra colors.
- Each module should have one title plus 0-2 short labels.
- Keep Chinese labels concise and evidence-driven.
- Keep secondary bilingual text visibly subordinate.

## Scientific standard

- Do not invent mechanisms, numbers, instrument details, architecture details, molecules, structures, or effects.
- Mark uncertain items as schematic.
- Every visible scientific claim must be traceable to user input or source material.

## Delivery standard

Deliver PPTX and, when requested, JSON spec plus quality report. The PPTX should remain fully editable after opening in PowerPoint.


## Information-density standard

- Choose a density profile before generation: `compact`, `standard`, or `rich`.
- Keep semantic information units within the profile range.
- Use text to name objects and claims, not to explain the whole method.
- Add visual evidence, result tags, or mechanism objects when the figure is too empty.
- Merge modules, shorten labels, or move detail to notes when the figure is too dense.
- Fail strict review when the figure has no clear evidence cue, no dominant information hierarchy, or visible text exceeds the selected profile budget.
