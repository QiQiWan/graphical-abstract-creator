# CyberPPT-Level Quality Standard

Use this standard for strict graphical abstract generation.

## Story standard

- One central visual claim.
- No decorative elements that do not support the claim.
- Every block must have a semantic role: input, method, mechanism, result, or application.
- Avoid turning the figure into a project-management workflow unless the paper itself is a workflow paper.

## Visual standard

- Use alignment grids and consistent spacing.
- Use native PowerPoint vector shapes only.
- Use restrained colors and strong hierarchy.
- Use callouts only for verified result or key mechanism.
- Avoid crowded labels, gradients, shadows, and clip-art aesthetics.

## Scientific standard

- No invented mechanisms.
- No invented numbers.
- No fake instrument diagrams.
- No non-source architecture details.
- Mark uncertain items as schematic.

## Delivery standard

Deliver PPTX and, when requested, JSON spec plus quality report. The PPTX should remain fully editable after opening in PowerPoint.
