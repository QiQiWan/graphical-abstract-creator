# Preview-First Workflow

The preview image is an intermediate design lock. It is not the final output unless the user explicitly requests preview only.

1. Read the content brief.
2. Confirm the compressed style card.
3. Build the master preview prompt.
4. Generate the preview image.
5. Audit preview hard gates.
6. If the user requested final creation or PPT output, continue to editable PPT reconstruction when the preview passes.
7. Build a reconstruction blueprint from the accepted preview.
8. Rebuild the editable PPT using native PowerPoint objects.
9. Validate preview-PPT consistency.
10. Deliver PPTX.
